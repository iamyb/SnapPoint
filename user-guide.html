Installation

SnapPoint is available on the Chrome Web Store and Microsoft Edge Add-ons. Choose the store that matches your browser.

Chrome

Google Chrome

Visit the Chrome Web Store listing and click Add to Chrome. The extension is installed immediately — no restart required.

Edge

Microsoft Edge

Visit the Edge Add-ons page and click Get. The extension activates on all your open tabs right away.

💡
After installing, click the SnapPoint icon in your browser toolbar to open the side panel and complete the one-time model setup.
🧭

Model Setup

SnapPoint supports both local models (via Ollama or LM Studio) and hosted API providers. Configure them from Settings → Model Settings.

Local Models

Running a model locally keeps all your data on-device — nothing leaves your machine. SnapPoint connects to a locally running inference server over HTTP.

1

Start your local inference server

Install Ollama or LM Studio, then pull the model you want — for example ollama pull qwen2.5:7b.

2

Set the endpoint in SnapPoint

Open Settings → Model Settings. Set the base URL to http://localhost:11434 (Ollama default) and enter the model name exactly.

3

Test the connection

Click Test Connection. A green checkmark confirms SnapPoint can reach your local server.

💡
For image-aware summaries, choose a multimodal model such as llava or qwen2-vl. SnapPoint will automatically use vision capabilities when the selected content contains images.

API Providers

If you prefer a hosted model, SnapPoint supports the following providers. Add your API key in Settings → Model Settings.

🔐
API keys are stored locally in your browser's extension storage and are never sent to SnapPoint's servers. See the Privacy Policy for details.

Quick Start

SnapPoint works on any page. The fastest path from page to summary is a single mouse drag or a click on a content block.

Using the Snap Pill

When you select text by dragging your mouse, a Snap pill appears next to your selection. Clicking it immediately sends the selected content to the active agent for summarization.

1

Drag to select content

Highlight any portion of a page — a paragraph, a comment thread, a code block — using your mouse.

2

Click the ✨ Snap pill

A small pill button appears at the edge of your selection. Click it to trigger summarization. The side panel opens automatically.

3

Read the summary

The side panel streams the result in real time. Key points, reasoning, and the agent type are shown at the top of the summary.

ℹ️
You can also click the ▲ expand arrow on the Snap pill to widen the selection to the parent DOM block before summarizing — useful for grabbing a full article section or comment thread.

Reading the Side Panel

The side panel shows three areas for every summary:

A

Status & Content Type

Shows the detected content category (e.g., Online Discussion, Research Paper, News Article) and which agent is being used.

B

Reasoning Block

Displays the model's live reasoning chain as it processes the content. This block collapses after the summary is complete.

C

Summary Output

The structured result, formatted as bullet points with bolded labels — ready to scan in seconds. Long summaries are scrollable.

Agents & Rules

An agent is a pre-configured summarization strategy — a prompt template plus optional post-processing rules. SnapPoint automatically selects the best agent for each page, but you can always override it.

Built-in Agents

Paper Agent

📄 Academic Papers

Extracts contribution, methodology, and limitations from arXiv and similar paper sites.

Forum Agent

💬 Discussions

Follows thread structure on HN, Reddit, and forums to surface arguments and consensus.

News Agent

🗞️ News Articles

Condenses long-form reporting into a brief with core facts and practical takeaway.

Repo Agent

📦 GitHub Repos

Summarizes README, purpose, and structure so you can quickly judge a project.

Finance Agent

💹 Market Updates

Turns earnings stories and macro headlines into an investor-focused brief.

General Agent

🌐 General Pages

A versatile fallback agent for any page that doesn't match a specialized category.

Custom Agents

You can create your own agents in Settings → Prompt Settings. Each custom agent has a name, a trigger pattern (URL match or keyword), and a prompt template.

1

Open Prompt Settings

Navigate to Settings → Prompt Settings and click New Agent.

2

Define a trigger

Enter a URL pattern (e.g., *.substack.com/*) or a content keyword that activates this agent automatically.

3

Write your prompt template

Use {{content}} as the placeholder for the selected text. Add instructions that reflect the tone, format, or focus you need.

4

Save and test

Visit a matching page, select some content, and click the Snap pill to verify the output.

# Example custom agent prompt for technical blogs You are a senior engineer reviewing a technical blog post. Summarize the following content in 3–5 bullet points. Focus on: the core technique, any trade-offs mentioned, and practical applicability. Content: {{content}}

Knowledge History

Every summary you generate is automatically saved to your local history. Access it anytime from the side panel or from Settings → History.

1

Browse history

Open the side panel and click the History tab to see a reverse-chronological list of all past summaries grouped by date.

2

Search summaries

Use the search bar at the top of the History view to find past summaries by keyword, URL, or content type.

3

Export or clear

From Settings → Storage you can export your history as a JSON file or clear selected entries to free up local storage.

💡
History is stored entirely in your browser's local extension storage. It is never synced to a server, and it persists across browser restarts.

FAQ

The Snap pill doesn't appear when I select text. What's wrong?
Some sites use user-select: none CSS to block text selection. Try using the DOM picker instead: right-click a content block and choose Snap with SnapPoint from the context menu, or use the side panel to trigger a full-page summary.
Can I use SnapPoint without an internet connection?
Yes — if you configure a local model (Ollama or LM Studio), SnapPoint works entirely offline. The extension itself never requires a connection; only the model backend needs to be reachable.
How does automatic agent detection work?
SnapPoint inspects the page's URL, document title, and a lightweight DOM signature to classify the content. The classification is done locally on your machine — no data is sent to detect the content type.
Can I summarize images embedded in an article?
Yes, if your configured model supports multimodal input (e.g., llava, qwen2-vl, or gpt-4o). SnapPoint will encode the image and include it in the prompt automatically. For text-only models, an OCR fallback extracts visible text from images before summarization.
How do I update or delete a custom agent?
Go to Settings → Prompt Settings, find the agent in the list, and click the edit or delete icon next to it. Changes take effect immediately on the next summarization.
Is my API key safe?
Your API keys are stored in the browser's extension storage, which is sandboxed to SnapPoint and inaccessible to other extensions or web pages. Keys are transmitted only to the AI provider endpoint you configured — never to SnapPoint's servers. See the Privacy Policy for full details.
